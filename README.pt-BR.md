## Por que este projeto existe

Muitos projetos de portfólio parecem desconectados da realidade.

Normalmente eles começam de prompts genéricos como “faça um CRUD” ou “crie um dashboard”, mas software do mundo real costuma nascer de algo mais concreto: falta de visibilidade, coordenação fraca, informação fragmentada, pessoas sobrecarregadas, atrasos recorrentes, falhas de acompanhamento ou sistemas que só funcionam porque alguém está o tempo todo compensando suas falhas.

Este repositório existe para documentar esse tipo de problema de uma forma que seja:

- humana
- descritiva
- tecnicamente útil
- amigável para opensource

A ideia é ajudar desenvolvedores a construir coisas mais próximas de produtos reais, ferramentas internas, sistemas operacionais e software orientado a comunidades.

---

## O que você vai encontrar aqui

Este repositório inclui:

- descrições de problemas do mundo real
- categorias baseadas em contextos operacionais reais
- relatos de problema em primeira pessoa
- requisitos técnicos para orientar possíveis soluções
- sugestões de MVP
- inspiração para projetos opensource com utilidade prática

---

## O que torna este repositório diferente

Este **não** é uma lista aleatória de ideias de app.

É uma coleção curada de problemas que:

- parecem reais
- afetam pessoas, times, negócios ou comunidades de verdade
- têm profundidade técnica relevante
- podem inspirar trabalho opensource útil
- vão além de ideias rasas como “clone de X” ou “faça um app básico”

Em vez de dizer:

- crie um CRM
- faça um dashboard
- monte um to-do app

Este repositório tenta mostrar:

- por que as pessoas perdem visibilidade
- por que a coordenação falha
- por que o acompanhamento se perde
- por que as decisões ficam reativas
- por que tantas rotinas dependem de memória e improviso
